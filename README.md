# آزمون ارشد (Android)

اپلیکیشن اندرویدی آزمون چهارگزینه‌ای کارشناسی ارشد ایران — با ساختار پروژه `market-prices` (صفحه HTML فارسی داخل شِل WebView کاتلینی).

## امکانات

- **بانک سؤالات جامع** بر اساس منابع و سرفصل‌های کنکور ارشد: دروس عمومی (زبان عمومی، استعداد تحصیلی) + رشته‌های مهندسی کامپیوتر، مدیریت، روانشناسی و حقوق — ۱۳ آزمون و ۲۰۰ سؤال، همگی با پاسخ تشریحی
- **آزمون زمان‌دار** با تایمر شمارش معکوس، ناوبری بین سؤالات، علامت‌گذاری برای مرور و پایان خودکار با اتمام وقت
- **نمره‌دهی مطابق کنکور سراسری**: هر ۳ پاسخ غلط یک پاسخ صحیح را خنثی می‌کند (نمره منفی یک‌سوم)
- **کارنامه کامل**: درصد کل، تفکیک صحیح/غلط/بی‌پاسخ، درصد به تفکیک هر درس و پیام تراز
- **مرور پاسخ‌ها** با نمایش گزینه صحیح، پاسخ کاربر و توضیح تشریحی هر سؤال
- **تاریخچه کارنامه‌ها** (ذخیره محلی) و نمایش بهترین درصد هر آزمون
- **ناوبری پایین** (مانند اپ قیمت دلار و طلا): خانه، کارنامه‌ها، آمار، بیشتر + دکمه وسط «آزمون سریع» (۲۰ سؤال تصادفی ترکیبی از همه رشته‌ها)
- **تب آمار**: تعداد آزمون‌های داده‌شده، میانگین و بهترین درصد + وضعیت هر درس
- **تب بیشتر**: تغییر تم، نسخه اپلیکیشن و کپی‌رایت (مانند market-prices)
- **بروزرسانی درون‌برنامه‌ای** (مانند market-prices): بررسی خودکار نسخه از GitHub؛ بروزرسانی محتوا (HTML) بدون نصب مجدد، یا دریافت و نصب APK کامل با نوار پیشرفت
- تم تیره/روشن، فونت وزیر، آزمون‌دهی کاملاً آفلاین و راست‌به‌چپ

## نیازمندی‌ها

- Node.js 20+
- ابزارهای بیلد اندروید (JDK 17 + Android SDK + کش Gradle) به‌صورت خودکار از
  `../market-prices/android/.tools/` به اشتراک گرفته می‌شوند. در صورت نبود، یک بار
  `bootstrap-android-deps.sh` پروژه market-prices را اجرا کنید.

## دستورات

```bash
# ساخت azmoon-arshad.html
npm run build

# ساخت APK امضاشده/دیباگ → android/azmoon-arshad.apk
npm run build:apk

# انتشار نسخه جدید (bump نسخه + بیلد + آپلود APK به GitHub Release + انتشار متادیتا)
npm run release:apk
# فلگ‌ها: --no-bump --no-build --no-upload --no-push
```

## ساختار پروژه

```
android/                      # پروژه Gradle (شِل WebView)
assets/fonts/                 # فونت Vazir-FD.ttf
tools/azmoon-arshad/
  data/                       # بانک سؤالات (هر گروه/رشته یک فایل)
  js/app.js                   # منطق UI (خانه، آزمون، کارنامه، مرور، تاریخچه)
  css/styles.css              # استایل‌ها (تم تیره/روشن)
  js/update-bridge.js         # پل بروزرسانی درون‌برنامه‌ای (شیت بروزرسانی + صدا + پیشرفت)
scripts/
  android-src/                # سورس Kotlin + AndroidManifest (سینک به android/)
                              # MainActivity + AzmoonUpdater + ApkUpdater + ApkInstallReceiver
  android-app-build.gradle.kts
  android-apk-version.json    # نسخه APK (versionCode/versionName) — با release:apk بامپ می‌شود
  build-azmoon-page.mjs       # ساخت HTML تک‌فایلی + اعتبارسنجی بانک سؤالات
  build-azmoon-apk.mjs        # ساخت APK (تزریق نسخه + کپی مانیفست نسخه به assets)
  publish-azmoon-apk.mjs      # انتشار: GitHub Release + متادیتای بروزرسانی روی main
```

## سازوکار بروزرسانی

دقیقاً مانند market-prices:

- اپ هر ۵ دقیقه (و ۱۲ ثانیه بعد از باز شدن) فایل
  `azmoon-arshad-app-version.json` را از `raw.githubusercontent.com` می‌خواند.
- اگر `apkVersionCode` جدیدتر باشد → شیت «بروزرسانی اپلیکیشن»: دانلود APK از GitHub Release
  (با اعتبارسنجی SHA-256) و نصب از طریق `PackageInstaller`.
- اگر فقط محتوا (HTML) جدیدتر باشد → شیت «بروزرسانی محتوا»: دانلود HTML + فونت در
  `filesDir` و بارگذاری مجدد بدون نصب.
- انتشار با `npm run release:apk` انجام می‌شود (نیازمند `gh auth login`).

## افزودن سؤال/آزمون جدید

در `tools/azmoon-arshad/data/` فایل گروه مربوطه را ویرایش کنید (یا گروه جدیدی بسازید و در
`data/index.mjs` اضافه کنید). ساختار هر سؤال:

```js
{
  subject: "نام درس",
  q: "متن سؤال",
  options: ["گزینه ۱", "گزینه ۲", "گزینه ۳", "گزینه ۴"],
  answer: 0,               // اندیس گزینه صحیح (۰ تا ۳)
  explanation: "پاسخ تشریحی",
  reading: "r1",           // اختیاری: ارجاع به متن درک مطلب در exam.readings
}
```

اسکریپت بیلد قبل از ساخت، بانک سؤالات را اعتبارسنجی می‌کند (۴ گزینه، پاسخ معتبر، تشریح، ...).

## امضای انتشار

بدون `android/keystore.properties`، خروجی با امضای دیباگ ساخته می‌شود. برای امضای انتشار،
مانند market-prices فایل keystore بسازید و `keystore.properties` را در `android/` قرار دهید.
